# 更新日志 (CHANGELOG)

## [v1.2.6] - 2026-03-08
### 修复
- **面试官“永不掉线”计划**：新增了面试引擎自动降级机制。当 Gemini 遇到 429 或 503 错误时，系统将自动切换至 DeepSeek 引擎继续面试。
- **面试逻辑增强**：强化了面试官的主动提问意识，确保在面对“下一题”指令时能迅速切换话题，且任何情况下都不会停止回答。

## [v1.2.5] - 2026-03-08
### 修复
- **参考资料终极保底方案**：新增了基于公司名称的自动链接生成器。如果 AI 联网搜索失败，系统将自动生成 Google、百度、企查查、领英的直达搜索链接，确保参考资料栏永不为空。

## [v1.2.4] - 2026-03-08
### 优化
- **参考资料抓取增强**：引入了“三重保险”链接提取算法（结构化提取 + 正则文本挖掘 + 搜索入口保底），解决了部分情况下参考资料显示为空的问题。

## [v1.2.3] - 2026-03-08
### 修复
- **环境变量多重回退机制**：重构了 API Key 获取逻辑，同时支持 `import.meta.env` 和 `process.env`，并兼容 `VITE_` 前缀和原始名称。
- **原因**：解决了在某些部署环境下环境变量无法被前端识别的问题。

## [v1.2.2] - 2026-03-08
### 修复
- **环境变量兼容性**：将 `DEEPSEEK_API_KEY` 更名为 `VITE_DEEPSEEK_API_KEY`，并修正了前端读取逻辑，解决了在 Netlify 部署后分析功能卡死的问题。

## [v1.2.1] - 2026-03-08
### 优化
- **参考资料直连**：优化了联网搜索的链接提取逻辑，现在右侧“参考资料”栏会直接展示可点击跳转的原始网页链接。
- **搜索深度增强**：在 Gemini 搜索阶段增加了官方网站、财务动态和员工评价的专项搜索，使分析背景更扎实。

## [v1.2.0] - 2026-03-08
### 新增
- **双引擎架构**：
  - **Gemini Engine**：负责实时联网搜索（获取公司背景）和面试聊天（快速响应）。
  - **DeepSeek Engine**：负责核心的简历深度分析、JD 匹配和面试题生成。
- **严谨性约束**：在 System Prompt 中强制执行“零幻觉、零夸大、零谄媚”原则。
- **时间线对齐**：所有 AI 逻辑均已对齐当前系统时间（2026-03-08）。

### 优化
- **分析流程重构**：采用“Gemini 搜索 + DeepSeek 分析”的组合拳，提升分析的深度与准确度。
- **降级保护**：如果 DeepSeek 调用失败，系统会自动回退到 Gemini 进行分析，确保服务不中断。

## [v1.1.1] - 2026-03-08
### 优化
- **模型回滚**：将分析模型从 `gemini-3.1-pro` 回滚至 `gemini-3-flash-preview`。
- **原因**：Pro 模型在免费层级的配额过于严格（429 错误），Flash 模型提供更稳定的并发支持和更快的响应速度。

## [v1.1.0] - 2026-03-08
### 新增
- **错误处理机制**：在分析页面增加了错误捕获和 UI 提示，避免 API 调用失败时页面空白。
- **自动降级逻辑**：在 `geminiService` 中增加了重试机制，如果带搜索的分析失败，会自动切换到标准模式。
- **版本记录系统**：正式启用 `CHANGELOG.md` 记录项目演进。

### 优化
- **AI 模型升级**：将分析模型从 `gemini-3-flash` 升级为 `gemini-3.1-pro-preview`，提升分析深度。
- **提示词工程**：优化了面试官的性格设定和分析报告的 Markdown 结构。

### 修复
- 修复了在网络波动或 API 限制下，匹配分析部分可能不显示内容的 Bug。
